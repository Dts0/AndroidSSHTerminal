## 5. 真实 SSH 连接（含 host key 确认）

### 5.1 网络可达性（前置）

| 测试 | 结果 |
|------|------|
| `curl -sSI https://dreamoftime0.com` | HTTP 200（Apache/2.4.58 Ubuntu）|
| `nslookup dreamoftime0.com` | 47.245.39.32（IPv4 解析成功）|
| `bash /dev/tcp/dreamoftime0.com/22` | OPEN（22 端口可达）|
| `echo password \| ssh test@...` | 登录成功（`whoami` → `test`）|

### 5.2 应用内首次连接（host key 弹窗）

点击列表项后弹出 **AlertDialog**：

| 元素 | 内容（已脱敏摘要）|
|------|-----|
| 标题 | 确认主机指纹 |
| 消息 | "即将连接 dreamoftime0.com:22\n算法：ecdsa-sha2-nistp256\n指纹：<SHA-256 hex>\n\n请确认该指纹与服务器端一致后再继续。" |
| 按钮 | 取消 / 信任并连接 |

**关键修复点验证**：
- ✅ **C1**：弹窗触发，host key 变更会**同样触发**（不静默接受）
- ✅ **M4**：算法正确解析为 `ecdsa-sha2-nistp256`（修复前会错误回退到 `ssh-rsa`）
- ✅ **M5**：算法 + 指纹组合匹配（不仅比对 fingerprint）
- ✅ **M6**：弹窗由 `StrictHostKeyRepository.lastRejectedKey` 驱动，**不依赖 JSch 异常文本**

### 5.3 信任后重连

点击"信任并连接"后：
- `lastRejectedKey` 写入 `trusted_host_keys.tsv`（m1 修复：host id 已被 SHA-256 哈希）
- 重建 SSH 连接，使用与展示给用户**同一**指纹（C2 TOCTOU 防御）

### 5.4 状态确认

- `tv_status = "已连接"`（绿色圆点 + 文字）
- `btn_connect` enabled=false / `btn_disconnect` enabled=true
- 软键盘自动弹出（`InputMethodManager.showSoftInput()`）

## 11. 已知限制与建议

1. **自动化字符输入限制**：`adb shell input text` 在 IME 激活状态下，部分字符会因 IME 拦截未到达 SSH。可使用 `input keyevent <ASCII>` 逐字符发送（更可靠但慢）。
2. **`screencap` 失效**：因 `FLAG_SECURE` 生效，需要用 `uiautomator dump` 读 UI 树，或通过 `PrintWindow` 截 Windows 窗口（绕过 secure flag）。
3. **真实键盘栏长按测试未做**：例如长按 Ctrl 切换"按住 Ctrl"模式（Termux 风格）。
4. **Session 重连超时未测**：手动 SIGSTOP SSH 服务端 30 秒后看 serverAliveInterval 行为。
5. **Host key 旋转未测**：手动修改 server 的 ssh_host_ecdsa_key 后看 HostKeyChanged 警告。
